프로젝트 개요: Broom 🧹

문제 정의

Windows 데스크탑 사용자는 '다운로드'나 '바탕화면' 폴더에 누적되는 수천 개의 파일을 수동으로 관리해야 하는 비효율을 겪습니다. 기존 클리너 앱은 스캔 속도가 느리거나, 사용자가 신뢰할 수 없는 '블랙박스'처럼 동작하는 문제가 있었습니다. 또한, 파일 및 데스크탑 지식이 부족한 사용자에게 친화적이지 않으며 파일에 큰 에러가 일어나기도 했습니다.

아키텍처 / 설계

성능과 지능형 기능이라는 두 마리 토끼를 잡기 위해 하이브리드 아키텍처를 설계했습니다.

  • 고성능 코어 (C++): 파일 시스템 스캔처럼 속도가 생명인 부분은 pybind11과 C++17 std::filesystem을 활용한 네이티브 모듈로 구현했습니다. [cite: analyzer_module.cpp]
  • 지능형 서비스 (AI): ai_services.py를 중앙 허브로 두어, 파일 정리, 로그 분석 등 모든 기능이 Gemini API를 호출하도록 설계했습니다. [cite: ai_services.py]
  • 백그라운드 자동화 (Python): watchdog과 scheduler 스레드가 메인 GUI와 독립적으로 동작하며 자동화 작업을 수행합니다. [cite: watcher_thread.py]

핵심 기능

C++ 고성능 스토리지 분석

비동기 네이티브 모듈 연동

Python의 os.walk는 대용량 드라이브 분석 시 GUI를 멈추게 합니다. 이를 해결하기 위해 C++17 std::filesystem과 pybind11을 활용한 네이티브 스캔 모듈을 구현했습니다. 이 모듈은 Python의 queue를 C++에서 직접 받아, 메인 스레드를 차단하지 않고(non-blocking) 스캔 결과를 실시간 스트리밍합니다.

AI 기반 지능형 자동화

Gemini API 기반 로그 분석 및 정리

단순 규칙을 넘어 Gemini API를 활용한 '지능형' 기능을 구현했습니다.

  • AI 파일 정리: "스크린샷만 모아줘" 같은 커스텀 명령을 AI가 JSON 계획으로 변환합니다.
  • AI 로그 필터링: AI가 캐시 경로 패턴을 학습하여 불필요한 로그를 영구 무시합니다.
  • AI 로그 요약: 수집된 로그 세션을 AI가 분석하여 "무슨 일이 있었는지" 요약합니다.

배운 점

Python GUI 앱에서 C++의 속도와 tkinter의 편의성을 결합하는 하이브리드 개발 방식을 익혔습니다. 또한, 메인 UI 스레드를 차단하지 않는 백그라운드 서비스를 구현하는 방법을 깊이 이해했습니다. Gemini API의 프롬프트 엔지니어링을 통해 '캐시 경로 추론'이나 '로그 요약' 같은 복잡한 작업을 안정적인 JSON 포맷으로 유도하는 노하우를 습득했습니다.
